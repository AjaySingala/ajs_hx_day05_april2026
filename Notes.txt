LLMs have parameters:
Temperature:
	- between 0 and 1.
	- For e.g.; 0 0.1 0.3 0.7 0.9 1
	- Determines the "creativity" of the answer to be generated
	- lower the number, the more deterministic
		- for the same question, same answer.
		- FAQs
	- higher the number, the more creative it will be.
		- ads, blogs, emails
		
top-p: 0 - 1 - probablity 
	- top-p: 0.9 = 90%
	- consider only words that form the top 90% of the list

How do we optimize token utilization?
- Structured output:
	- in 3-5 bullet points
	- in less than 500 words
- How you write your prompts
	- Don't write very long prompts every time.
	- do not provide context, info that is not really relevant
		- confuse the LLM
	- You don't have specify a "role" every time.
- using caching
	- Q&A / FAQ:
		- queries are more or less the same.
		- When a query is submitted first time
			- fetch the results from the LLM
			- store the result in cache
		- next time when the same query is submitted
			- get it from the cache
			- don't even send the query to the LLM
		- control how long will the info be available in the cache

RAG vs Fine-tuning
	- more or less static data (does not change frequently, changes very rarely)
		- Fine-tuning
	- data that can change regularly (not necessarily daily)
		- policies, SOPs, work order templates, contracts, coding standards
		- Leave policy may change every year
		- RAG (Documents)
	- Live data
		- NEITHER RAG NOR Fine-tuning
		- collect this live data, then use it to retrain your model
		
AI Agent vs Agentic AI
	- AI Agent - singular - a single agent completing a specific task
	- Agentic AI: >1 agents participating in a "proces" (worfklow) to achieve a task
